# BACKEND_PROGRESS

Last updated: 2026-05-31

## Current Backend Path

- API project: `backend/ContactApi`
- Solution: `Portfolio.sln`
- Local database: Docker Compose Postgres from `docker-compose.yml`

## Current Status

The backend is currently a .NET 8 ASP.NET Core Minimal API with:

- a working `GET /api/health` endpoint
- PostgreSQL configured through EF Core and Npgsql
- a `ContactSubmission` entity
- an `AppDbContext`
- an initial EF Core migration for `contact_submissions`
- local Postgres provided by Docker Compose on host port `5433`
- automatic database migration on app startup, except when `ASPNETCORE_ENVIRONMENT=Testing`

The contact form submission endpoint is not implemented yet. There is currently no tracked `POST /api/contact` endpoint, validator, contact service, email sender, Turnstile verifier, or test project source.

## What Has Been Built So Far

### 1. Project baseline

- `backend/ContactApi/ContactApi.csproj` targets `net8.0`.
- Nullable reference types are enabled.
- Implicit usings are enabled.
- User secrets are initialized through `UserSecretsId`.

Installed backend packages:

- `Microsoft.EntityFrameworkCore` `8.0.11`
- `Microsoft.EntityFrameworkCore.Design` `8.0.11`
- `Npgsql.EntityFrameworkCore.PostgreSQL` `8.0.11`
- `EFCore.NamingConventions` `8.0.3`
- `FluentValidation` `11.10.0`
- `FluentValidation.DependencyInjectionExtensions` `11.10.0`

### 2. Program wiring

`backend/ContactApi/Program.cs` currently:

- registers `AppDbContext`
- reads the database connection string from `ConnectionStrings:Default`
- uses PostgreSQL through Npgsql
- applies snake_case database naming conventions
- runs `db.Database.Migrate()` on startup outside the `Testing` environment
- maps `GET /api/health`

Current health response:

```json
{"ok":true,"service":"ContactAPI"}
```

### 3. Local PostgreSQL with Docker Compose

`docker-compose.yml` defines a local Postgres 16 container:

- container name: `contactapi`
- database: `contactapi`
- username: `postgres`
- password: `devpassword`
- host port: `5433`
- container port: `5432`
- data volume: `pgdata`

The local development connection string uses port `5433` because port `5432` was already in use locally.

### 4. Development configuration

`backend/ContactApi/appsettings.Development.json` currently contains:

- logging configuration
- `AllowedOrigins` set to `http://localhost:5173`
- `ConnectionStrings:Default` pointing to local Docker Postgres on port `5433`
- placeholder Resend email configuration
- Cloudflare Turnstile verify URL

Resend and Turnstile are configured as settings only right now. Their service implementations are not built yet.

### 5. Database model

`backend/ContactApi/Models/Entities/ContactSubmission.cs` defines the persisted contact submission row with:

- `Id`
- `Name`
- `Email`
- `Message`
- `SubmittedAt`
- `IpAddress`
- `UserAgent`
- `EmailSentAt`
- `EmailError`

### 6. EF Core context

`backend/ContactApi/Data/AppDbContext.cs` defines:

- `DbSet<ContactSubmission> ContactSubmissions`
- table name: `contact_submissions`
- UUID primary key generated by `gen_random_uuid()`
- required `name`, `email`, and `message`
- max length `255` for `name` and `email`
- max length `512` for `user_agent`
- default `submitted_at` value from `NOW()`
- index on `submitted_at`

### 7. Initial migration

The initial migration exists under:

```text
backend/ContactApi/Data/Migrations/
```

It creates the `contact_submissions` table and the `submitted_at` index.

### 8. Launch profile

`backend/ContactApi/Properties/launchSettings.json` defines an HTTP profile:

- URL: `http://localhost:5236`
- environment: `Development`
- browser launch: disabled

### 9. Build artifact protection

`.gitignore` ignores .NET build output:

- `**/bin/`
- `**/obj/`

The generated `bin/` and `obj/` folders should not be committed.

## How To Run Locally

### 1. Start Postgres

From the repo root:

```bash
docker compose up -d
```

Verify the container:

```bash
docker exec contactapi pg_isready -U postgres
```

Expected:

```text
/var/run/postgresql:5432 - accepting connections
```

### 2. Build the API

```bash
dotnet build backend/ContactApi/ContactApi.csproj
```

Latest verification on 2026-05-31:

```text
Build succeeded.
0 Warning(s)
0 Error(s)
```

Note: the local machine may print a macOS `CSSM_ModuleLoad()` message during restore/build. It is not counted as a .NET build warning.

### 3. Run the API

```bash
dotnet run --project backend/ContactApi/ContactApi.csproj
```

The app should start on:

```text
http://localhost:5236
```

On startup, the API applies EF Core migrations automatically when not running in the `Testing` environment.

## How To Stop Local Services

### Stop the API

If running in the current terminal:

```text
Ctrl + C
```

If a detached process is using the API port:

```bash
kill -9 $(lsof -ti :5236)
```

### Stop Postgres

```bash
docker compose down
```

To stop Postgres and delete the local database volume:

```bash
docker compose down -v
```

Use `-v` carefully because it deletes the local `pgdata` database volume.
